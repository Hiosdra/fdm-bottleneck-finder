# Contributing to 3D Printer Speed Bottleneck Finder

Thank you for your interest in contributing to the 3D Printer Speed Bottleneck Finder! We welcome contributions from the community.

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion for improvement:

1. Check if the issue already exists in the [Issues](https://github.com/Hiosdra/fdm-bottleneck-finder/issues) section
2. If not, create a new issue with a clear title and description
3. Include steps to reproduce (for bugs) or use cases (for features)
4. Add relevant details like browser version, printer profiles used, etc.

### Submitting Changes

1. **Fork the repository** to your own GitHub account
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/YOUR-USERNAME/fdm-bottleneck-finder.git
   ```
3. **Create a new branch** for your changes:
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. **Make your changes** to the code
5. **Test your changes** thoroughly in a web browser
6. **Commit your changes** with clear, descriptive commit messages:
   ```bash
   git commit -m "Add feature: description of your changes"
   ```
7. **Push to your fork**:
   ```bash
   git push origin feature/your-feature-name
   ```
8. **Open a Pull Request** from your fork to the main repository

### Development Guidelines

- The project is a single-page HTML application with embedded JavaScript and CSS
- Test changes in multiple browsers (Chrome, Firefox, Safari) when possible
- Maintain the existing code style and formatting
- Keep the physics calculations accurate and well-commented
- Ensure the UI remains responsive on mobile devices

### Code of Conduct

- Be respectful and constructive in all interactions
- Focus on the technical merits of contributions
- Help create a welcoming environment for all contributors

## Questions?

If you have questions about contributing, feel free to:

- Open an issue with the "question" label
- Start a discussion in the repository
